# 功能需求书（PRD）

**产品名（暂定）**：MailPilot Local  
**版本**：v1.0  
**平台**：macOS  
**定位**：本地优先（Local-first）的 AI 邮件助手，帮助用户快速处理邮件，不上传敏感正文到云端。

---

## 1. 背景与目标

### 1.1 背景
高管/创始人每天邮件量大，人工筛选和回复耗时，且对隐私敏感，不愿将邮件内容交给第三方 SaaS。

### 1.2 目标
在本机完成邮件“收集→优先级判断→摘要→回复草稿”闭环，显著减少用户处理邮件时间。

### 1.3 成功指标（v1）
- 用户每日节省处理邮件时间：**30分钟+**
- Top邮件识别准确主观满意度：**>70%**
- App 首次可用时间（安装到看到摘要）：**<10分钟**

---

## 2. 用户与场景

### 2.1 目标用户
- 创始人 / 高管 / 个体顾问
- 邮件负担重，重视隐私

### 2.2 核心场景
1) 早上打开 App，快速看今日最重要 5 封邮件  
2) 每封邮件看到 3 行摘要和建议动作  
3) 一键生成回复草稿，复制回邮箱发送

---

## 3. 版本范围（Scope）

### 3.1 v1 必做（MVP）
1. 邮箱接入（先 Gmail，OAuth）
2. 邮件列表拉取（最近 N 封）
3. Top 5 重点邮件排序
4. 单封邮件 3 行摘要
5. 建议动作标签（回复/稍后/归档）
6. 回复草稿生成（复制按钮）
7. 本地存储（SQLite）
8. 一键清空本地数据

### 3.2 v1 不做
- 直接发送邮件
- 多邮箱统一收件箱
- 团队协作
- 移动端 App
- 自动回信

---

## 4. 功能需求（详细）

### F1. 账户接入
- 支持 Gmail OAuth 登录
- 用户可断开账号
- Token 本地安全保存（Keychain）

### F2. 邮件拉取
- 拉取最近 7 天或最近 200 封（可配置）
- 字段：发件人、主题、时间、正文片段

### F3. 重点邮件排序
- 依据：发件人重要度、关键词、是否需行动、时间紧迫性
- 输出：Top 5 列表

### F4. 摘要与动作建议
- 每封生成：
  - 3 行摘要
  - 动作建议（回复/稍后/归档）
  - 建议优先级（高/中/低）

### F5. 回复草稿
- 提供 2 种语气：简洁专业 / 友好
- 一键复制到剪贴板
- 记录“草稿已复制”状态（仅本地）

### F6. 本地数据控制
- 用户可查看本地缓存大小
- 一键删除本地邮件缓存与历史摘要

---

## 5. 非功能需求

### 5.1 隐私与安全
- 默认不上传邮件正文到你方服务器
- 所有处理默认本地进行（或用户明确启用云模型）
- 支持“用户自带 API Key”模式
- 本地敏感数据加密存储（Keychain + DB加密可选）

### 5.2 性能
- 启动时间 < 3 秒
- 200 封邮件摘要生成 < 2 分钟（首次）
- 后续增量更新 < 20 秒

### 5.3 稳定性
- 邮件拉取失败可重试
- AI 失败时给出可读错误，不阻塞其他功能

---

## 6. 信息架构与页面

1. **登录页**
- Gmail 连接按钮
- 隐私说明入口

2. **收件总览页**
- Top 5 卡片
- “刷新”按钮
- 今日处理统计

3. **邮件详情页**
- 原文片段
- 3 行摘要
- 建议动作
- 生成回复草稿按钮

4. **设置页**
- 账号管理
- API Key 设置
- 数据清空
- 隐私策略

---

## 7. 验收标准（UAT）

- 能成功 OAuth 登录并拉取邮件
- 能稳定展示 Top 5
- 每封可生成摘要与动作建议
- 回复草稿可复制
- 一键清空本地数据生效
- 断网/接口失败时有明确错误提示

---

## 8. 开发里程碑（建议）

- **Week 1**：项目骨架 + 登录 + 邮件拉取
- **Week 2**：Top排序 + 摘要 + 草稿
- **Week 3**：设置页 + 数据清理 + 错误处理 + 打包演示
